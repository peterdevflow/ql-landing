# Landing Page Best Practices

> This document serves as guidelines for AI content generation and UX improvements.

---

## 1. Core Principles

| Principle             | Description                         | Implementation                                                                         |
| --------------------- | ----------------------------------- | -------------------------------------------------------------------------------------- |
| **Single Focus**      | One primary goal per page           | Related question for the content: "What is the ONE action you want visitors to take?"  |
| **5-Second Clarity**  | Visitor understands topic instantly | Hero section must convey core value proposition                                        |
| **30-Second Summary** | Page can be summarized quickly      | Limit sections, keep copy concise                                                      |
| **Source-Page Match** | Content matches traffic source      | SEO keywords align with page content                                                   |
| **Personality Match** | Tone reflects brand identity        | Related question for the content: "What tone of voice?" (professional/friendly/urgent) |

---

## 2. Content Structure

### Required Sections

| Section               | Guidelines              | Character Limits                   |
| --------------------- | ----------------------- | ---------------------------------- |
| **Hero Headline**     | Clear, benefit-focused  | Max 3 lines, ~60-80 chars          |
| **Hero Subheadline**  | Supports main headline  | Max 3 lines, ~100-120 chars        |
| **Problem Statement** | What pain do you solve? | 4-5 lines per paragraph            |
| **Solution/Benefits** | How life improves       | Bullet points preferred            |
| **Pricing**           | Clear, exact pricing    | 3 tiers recommended                |
| **CTA**               | Single, clear action    | Button text + optional explanation |

### Optional Sections (User-Configurable)

| Section                | When to Include           | Prompt Question                               |
| ---------------------- | ------------------------- | --------------------------------------------- |
| **Testimonials**       | If user has social proof  | "Do you have customer testimonials?"          |
| **FAQ**                | Complex services/products | "What questions do customers frequently ask?" |
| **About Us**           | Personal services         | "Do you want to include your story?"          |
| **Free Samples**       | Lead magnets available    | "Do you offer free trials/samples?"           |
| **Objection Handling** | High-ticket items         | "What concerns might visitors have?"          |

---

## 3. Persuasion Guidelines

### Text Quality Checklist

- [ ] Every headline has ONE main thought
- [ ] Every section contains relevant keywords
- [ ] Paragraphs are 4-5 lines maximum
- [ ] Remove unnecessary/filler text
- [ ] Benefits > Features in copy
- [ ] Address visitor objections proactively

### Trust Elements (Must Include)

- Contact information visible
- Business logo prominent
- Social proof (testimonials, logos, numbers)
- Clear pricing (no hidden costs)

---

## 4. UI/UX Requirements

### Typography

| Element     | Size    | Notes                    |
| ----------- | ------- | ------------------------ |
| Body text   | 16-24px | Never smaller than 16px  |
| Headlines   | 32-48px | Consistent hierarchy     |
| Subheadings | 20-28px | Clear visual distinction |

### Visual Design

| Requirement           | Implementation                         |
| --------------------- | -------------------------------------- |
| Block differentiation | Alternate background colors/shades     |
| Button visibility     | High contrast, prominent placement     |
| Line alignment        | Consistent throughout                  |
| Image quality         | Avoid overused stock photos            |
| Human presence        | Prefer images with people/interactions |
| Hero image match      | Must relate to main content            |

### Layout Rules

- No navigation menus (single-page focus)
- Minimal footer (only essential links)
- Mobile-first responsive design
- Consistent spacing and rhythm
